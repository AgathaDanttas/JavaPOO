# Estudos de Programação Orientada a Objetos (POO) em Java

Este repositório reúne exercícios práticos e estudos de modelagem de domínio desenvolvidos em **Java**, com foco no fortalecimento dos fundamentos de **Programação Orientada a Objetos** e na evolução da organização, clareza e responsabilidade do código.

## Visão geral

Este projeto foi construído como parte da minha jornada de aprendizado em desenvolvimento de software, com ênfase em escrever código orientado a objetos de forma mais clara, estruturada e coerente com as regras de negócio propostas.

Ao longo dos exercícios, trabalhei tanto com:

- **implementação de classes**
- **análise e modelagem de domínio**

Ou seja, além da prática de código, este repositório também reflete o processo de identificar entidades, atributos, comportamentos e responsabilidades a partir de enunciados e problemas descritivos.

---

## Principais conceitos praticados

- Fundamentos de Programação Orientada a Objetos
- Criação de classes e objetos
- Construtores
- Encapsulamento
- Separação de responsabilidades
- Definição e organização de métodos
- Clareza de nomenclatura
- Implementação de regras de negócio
- Validação de dados
- Modelagem de domínio

---

## Exercícios implementados

### Rectangle
Classe desenvolvida para representar um retângulo e realizar cálculos geométricos.

**Conceitos trabalhados:**
- definição de atributos
- inicialização via construtor
- cálculo de área e perímetro
- métodos de exibição de dados

---

### Employee
Classe criada para representar um funcionário com informações salariais e tributárias.

**Conceitos trabalhados:**
- cálculo de salário líquido
- aumento salarial com base em percentual
- alteração de estado do objeto
- distinção entre métodos que calculam, alteram e exibem informações

---

### Product
Exercício voltado para controle de estoque de produtos.

**Conceitos trabalhados:**
- manipulação de estoque
- validação de entrada
- verificação de regras antes de alterar dados
- retorno booleano para indicar sucesso ou falha em operações
- cálculo de valor total em estoque

---

### Modelagem de Sistema de Biblioteca
Exercício focado em análise e modelagem de domínio a partir de um texto descritivo.

**Entidades modeladas:**
- `Library`
- `Book`
- `User`
- `Loan`

**Conceitos trabalhados:**
- identificação de classes a partir de requisitos
- distribuição de atributos entre entidades
- definição de responsabilidades
- fluxo de empréstimo e devolução
- status de disponibilidade do livro
- data prevista e data real de devolução
- raciocínio sobre taxa por atraso

Essa parte do repositório representa não apenas a prática de implementação, mas também a capacidade de interpretar problemas e transformá-los em uma estrutura orientada a objetos mais organizada.

---

Este repositório reflete minha evolução em:
